RIGOR

How A-DAP-T analyses your AI agent for pre-deployment safety risks — without executing a single line of your code.

6 Risk Categories
ANALYSIS PIPELINE
01

Static File Analysis

A-DAP-T parses your uploaded project without executing it. Python source files, prompt text files, JSON/YAML config files, and tool definitions are all read as text and tokenised for pattern matching. No subprocess calls, no sandbox, no runtime.

AST PARSINGPROMPT TEXTYAML/JSON CONFIGNO EXECUTION
02

Rule-Based Risk Scanner

A curated ruleset of ~80 patterns is applied across all parsed files. Rules detect hardcoded secrets (regex + entropy), unguarded input concatenation, overly permissive tool schemas, missing human-in-the-loop checks, PII-leaking context builders, and absent logging calls. Each rule maps directly to one of six risk categories and produces a structured finding.

~80 RULESENTROPY ANALYSISPATTERN MATCHINGSTRUCTURED FINDINGS
03

Gemini-Assisted Explanations

For each raw finding, A-DAP-T calls the Gemini API to generate a plain-language explanation of why the issue matters in the context of your specific agent, and a concrete suggested fix. Gemini sees only the relevant code snippet and the rule description — not your full codebase or any secrets.

GEMINI APISNIPPET-SCOPEDWHY IT MATTERSSUGGESTED FIX
04

Controlled Attack Replay

A-DAP-T runs a set of known adversarial prompts against the agent's system prompt in a fully isolated, sandboxed inference call. This tests whether the system prompt is susceptible to role-override, prompt-echo, and jailbreak patterns. The agent is never connected to any real tools during replay — only the language model is invoked.

RISK CATEGORIES EXPLAINED
💉Prompt Injection Risk

Checks whether user-controlled input can override or extend the agent's system instructions. The most common and highest-impact risk in deployed agents.

Direct string concatenation of user input into prompt No input escaping or sanitisation layer present System prompt does not include injection-resistance instructions Attack replay: role-override and instruction-ignore patterns
🔑Secret Exposure Risk

Detects API keys, tokens, passwords, and other credentials embedded in source files or prompts rather than loaded from secure environment variables.

High-entropy strings matching known key formats (regex) Hardcoded OpenAI / Anthropic / AWS / GCP key patterns System prompt readable and echoed on direct request Credentials in YAML config files or .env committed to repo
🛠️Tool Permission Risk

Evaluates whether tools are scoped appropriately. Broad permissions amplify every other risk — an injected instruction can only do what the tools allow.

Tools accepting unrestricted file paths, URLs, or recipients Write or delete operations without confirmation gate No per-session tool invocation budget Tool schemas with wildcard or missing parameter constraints
🧑‍⚖️Human Approval Risk

Identifies high-impact actions — sending messages, modifying data, making purchases — that execute autonomously without a human confirmation step.

No approval gate before email, SMS, or webhook send calls Database write operations triggered without confirmation No interrupt/pause mechanism for multi-step plans Missing HITL (human-in-the-loop) annotation in agent config
📤Data Exposure Risk

Flags cases where sensitive or personal data is included in LLM context without redaction, or transmitted to external APIs unnecessarily.

PII patterns (email, phone, SSN) inserted into prompt context Full database records passed to external model without masking No context-length guard on retrieved documents Conversation history sent to third-party APIs without consent
📋Auditability Risk

Checks whether the agent's actions are observable, traceable, and explainable after the fact — a requirement for both incident response and compliance.

No structured logging around tool calls Errors swallowed silently without log emission No session or request ID propagated through calls LLM reasoning steps not captured or stored
WHAT A-DAP-T IS AND ISN'T
✓ A-DAP-T does
  • Static analysis of source files, prompts, and configs
  • Rule-based detection of known AI agent risk patterns
  • Gemini-assisted plain-language explanations per finding
  • Controlled attack replay against system prompts (no real tools)
  • Actionable remediation checklist with severity prioritisation
  • Safety score across six risk categories
✗ A-DAP-T does not
  • Execute uploaded code — analysis is purely static
  • Replace a full security audit or penetration test
  • Guarantee zero vulnerabilities in production
  • Test dynamic or runtime behaviour of your agent
  • Scan network infrastructure, APIs, or cloud environments
  • Store or retain uploaded project files after the scan session
